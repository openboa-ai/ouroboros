## Current Priorities

- keep the top-level product framing clear: `autokairos = automated weak-to-strong trader`
- treat PRD 1 through PRD 4 as the locked architecture input contract
- keep architecture downstream of PRDs rather than parallel to them
- keep the first lovable path centered on one serious solo crypto operator
- keep the first market fixed as Binance BTC perpetual futures while preserving adapter seams
- keep the end-to-end path `agent-originated hypothesis -> candidate -> evidence -> live gate -> live execution -> wake/intervention` continuous
- keep live autonomy full within explicit limits
- keep operator trust, wake, and intervention product-visible
- keep the active spec baseline narrow and tied to current PRD implementation need
- keep [wiki/product/mlp-01/07-implementation-plan.md](wiki/product/mlp-01/07-implementation-plan.md) as
  the canonical build-order document instead of old subsystem implementation plans
- keep [wiki/architecture/01-pr1-path-becomes-real-design.md](wiki/architecture/01-pr1-path-becomes-real-design.md)
  as the canonical PR1 implementation-shape document beneath the higher-level build-order plan
