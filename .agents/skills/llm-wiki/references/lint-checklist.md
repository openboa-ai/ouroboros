# Lint Checklist

Use this checklist when health-checking maintained project documents.

## Substance

- Are the project documents carrying knowledge, or only structure?
- Should some documents be merged rather than kept as thin fragments?

## Naming

- Is terminology consistent across maintained documents?
- Are old names still lingering after a rename?

## Content

- Are there duplicate documents covering the same concept?
- Are there contradictions between maintained documents?
- Are claims backed by provenance?

## Connectivity

- Does the index still reflect the main entry points?
- Are important documents missing from the index?

## Hygiene

- Is the execution ledger current?
- Are raw sources staying immutable?
