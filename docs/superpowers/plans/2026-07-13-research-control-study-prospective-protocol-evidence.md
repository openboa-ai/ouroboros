# ResearchControlStudy Prospective Protocol Evidence Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Execute and restart-replay one precommitted six-replication ResearchControlStudy through
real CandidateArena, paper provider, sandbox, arm, campaign, and study services.

**Architecture:** A checked-in, LocalStore-validated TradingReview graph supplies immutable input
state. The integration test copies it to a temporary root, precommits all replications, composes
arm-local sessions with real loopback providers and deterministic sandbox processes, advances a
controlled public-market clock, and verifies exact terminal evidence after restart.

**Tech Stack:** TypeScript, Vitest, LocalStore, CandidateArena, PaperTradingSessionService,
DeterministicSandboxAdapter, loopback HTTP paper provider.

## Global Constraints

- Treat fixture output only as protocol evidence, never economic or causal success.
- Commit all six replication identities before the first campaign effect.
- Preserve ordinary LocalStore graph validation; do not override promotion or evidence getters.
- Use public read-only BTCUSDT data, paper-only Gateway authority, and no private/live fields.
- Track and close every provider and sandbox process.
- Never stage `.superpowers/`.

Provide:

```ts
export function prospectiveClock(startAt: string) {
  let current = startAt;
  return {
    now: () => current,
    async sleep(milliseconds: number) {
      current = new Date(Date.parse(current) + milliseconds).toISOString();
      await new Promise<void>((resolve) => setImmediate(resolve));
    }
  };
}
```
