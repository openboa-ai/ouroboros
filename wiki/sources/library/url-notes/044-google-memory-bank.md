## What This Source Actually Proves

Memory Bank dynamically generates long-term memories from conversations between users and agents.
It stores personalized information across sessions, scopes memories by agent/user, and supports
memory extraction, consolidation, asynchronous generation, continuous event ingestion, customizable
topics/examples, and multimodal input.

For autokairos, memory is useful but dangerous. It is context, not evidence.

## Mechanisms To Preserve

| Mechanism | autokairos implication |
| --- | --- |
| Isolated memory collections | Candidate, operator, market, and runtime memories need scopes. |
| Memory as independent fact | Memories should be addressable artifacts, not opaque hidden context. |
| Extraction/consolidation | Memory generation should be traceable and reviewable. |
| Background generation | Memory can be produced asynchronously, but must not mutate evidence silently. |
| Topics/few-shot customization | Memory policy defines what is meaningful; it is a governance choice. |
| Event ingestion | Trace/events can feed memory, but memory remains derived context. |

## Deep autokairos Insight

This source clarifies memory architecture:

```text
Trace = what happened
Memory = selected/consolidated context derived from history
EvidenceRecord = externally judged result
PromotionDecision = governance action
```

If memory influences live behavior, autokairos must know which memory was used and whether it was
trusted, reviewed, or generated by the runtime.

## What Not To Copy

- Do not let memory auto-consolidation rewrite candidate truth.
- Do not treat generated memories as facts without provenance.
- Do not put secrets, evaluator ground truth, or exchange credentials in memory bank.
- Do not let memory improve personalization while hiding audit influence.

## Design Questions Forced By This Source

- What memory scopes exist for trader-system candidates?
- Which memories are read-only versus runtime-writable?
- How does memory become visible in trace?
- Can a candidate be promoted if it depends on unreviewed memory?

## autokairos Design Pressure

Memory must be explicit, scoped, auditable, and derived from trace, not a hidden authority layer.
