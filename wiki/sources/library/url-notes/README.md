# URL-Level Source Notes

This directory contains one maintained note per supplied URL.

Cluster notes remain useful for comparing a product family, but URL-level notes are the detailed
ingestion layer. Each note should answer:

- what the source actually proves
- what mechanism or boundary matters
- what autokairos should copy as a principle
- what autokairos must not copy literally
- what design question the source should force before implementation

Do not treat a URL as deeply ingested until its note has real design implications, not just a short
summary.
