# autokairos Roadmap: Now, Next, Later

## Purpose

This page separates current focus from later ambition.

It exists to prevent roadmap drift from weakening the first lovable proof.

## Now

Current product focus is:

- lock the PM document stack
- complete MLP-01 planning
- ship one lovable trader-system runtime proof on Binance BTC perpetual futures
- prove agent-built `TraderSystemCandidate` -> externally counted evaluation -> promotion decision
  -> bounded live `TraderSystemRuntime` -> intervention

Anything that does not strengthen that proof is not `Now`.

## Next

After the first lovable proof is credible, the next logical layer is:

- harden bounded live runtime operation
- improve candidate quality and promotion readability
- refine wake and intervention confidence
- tighten adapter seams for later venue portability
- improve CapabilityPackage packaging and reuse without turning marketplace into current scope

`Next` still serves the same wedge. It does not redefine the company.

## Later

Only after the first wedge is believable should autokairos consider:

- additional venues
- more instruments beyond Binance BTC perpetual futures
- broader portfolio-level coordination
- team and approval workflows
- wider external APIs

`Later` is a consequence of success, not a substitute for it.

## Not On The Current Roadmap

The following are explicitly not current roadmap commitments:

- multi-team workflow platform
- consumer investing assistant experiences
- generic quant research environment
- broad venue-agnostic marketing promises
- full CapabilityPackage marketplace as a first-class goal

## Roadmap Rule

If a proposed item does not strengthen:

- the lovable proof
- the serious solo operator workflow
- the governed candidate-to-runtime path

it should not enter `Now`.
